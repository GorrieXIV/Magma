End of File Marker






The I/O types below all need some way of indicating when a read request
fails due to no more data being available.  This is achieved by returning
a special "end of file" (shortened to "EOF") string that is not equal to
any normal string.



Eof() : -> MonStgElt



Creates the special EOF string.



IsEof(S) : MonStgElt -> BoolElt



Given a string S, return whether S is the special EOF string.



AtEof(I) : IO -> BoolElt



Given an I/O object I, returns whether all data is known to have been
read from I (and thus that further reads will return the special EOF
string).  Note that if this function returns false then it may still
be the case that the next read returns EOF; typically AtEof only
returns true when a previous read has already returned EOF.
